#### &#x09;			**RECURSION**



1. Recursion is a programming technique in which a function or method calls itself to solve a problem.
2. A recursive function breaks a large problem into smaller and simpler subproblems.



**Every recursive function must have:**

* Base Case – Stops the recursion.
* Recursive Case – Calls itself with a smaller problem.



**How Recursion Simplifies Problems**

* It reduces complex problems into smaller, manageable parts.
* It makes code shorter and easier to understand.
* It is useful for problems that naturally have repeating subproblems.
* It avoids writing lengthy loops for certain problems.



**Advantages**

1. Simple and elegant code.
2. Easier to solve complex problems.
3. Useful for hierarchical and repetitive tasks.



**Disadvantages**

1. Uses extra memory due to function call stack.
2. Can be slower than iterative solutions for large inputs.
3. May cause Stack Overflow if the recursion depth becomes too large.
