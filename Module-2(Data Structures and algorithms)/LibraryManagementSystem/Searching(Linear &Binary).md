###### **Linear Search**

* Linear Search checks each element one by one until the required element is found.
* It works on both sorted and unsorted data.
* Simple and easy to implement.
* Suitable for small datasets.





###### **Binary Search**

* Binary Search repeatedly checks the middle element of a sorted list.
* If the element is not found, it eliminates half of the remaining elements.
* Works only on sorted data.
* Suitable for large datasets.





###### **Comparison**

**Searching Method**

* Linear Search: Checks elements one by one.
* Binary Search: Checks the middle element and eliminates half of the data.



**Data Requirement**

* Linear Search: Works on both sorted and unsorted data.
* Binary Search: Works only on sorted data.



**Speed**

* Linear Search: Slower for large datasets.
* Binary Search: Faster for large datasets.



**Best Case Time Complexity**

* Linear Search: O(1)
* Binary Search: O(1)



**Average Case Time Complexity**

* Linear Search: O(n)
* Binary Search: O(log n)



**Worst Case Time Complexity**

* Linear Search: O(n)
* Binary Search: O(log n)
